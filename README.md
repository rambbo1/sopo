# 소포수령증 자동화 — GitHub 기존버전 기반 + v29 기능 반영

이 프로젝트는 `kanghyojeong2298/-_-` 기존 Streamlit 구조를 베이스로 하고, 최근 v29에서 정리한 환율 자동수집/문서선택 기능을 합친 버전입니다.

## 반영 기준

- PDF 파싱 및 매출집계 엑셀 생성: 기존 GitHub 방식 사용
  - `modules/pdf_parser.py`
  - `modules/excel_writer.py`
- 큐텐재팬 정보 입력: 기존 STEP 2 수동 입력 방식 유지
- 환율: 서울외국환중개 기간별 매매기준율 자동 수집 + `data/exchange_rate_cache.csv` 캐시
- 처리 시작: 매출집계 / 영세율첨부서류제출명세서 / 수출실적명세서 선택 생성
- 결과 다운로드: ZIP이 아니라 파일별 `.xlsx` 다운로드 버튼 표시

## 파일명 인식 규칙

| 파일명 패턴 | 플랫폼 |
|---|---|
| `유엠(UM)_MY_*.pdf` | 쇼피 말레이시아 |
| `유엠(UM)_PH_*.pdf` | 쇼피 필리핀 |
| `유엠(UM)_SG_*.pdf` | 쇼피 싱가폴 |
| `유엠(UM)_TH_*.pdf` | 쇼피 태국 |
| `유엠(UM)_TW_*.pdf` | 쇼피 대만 |
| `유엠(UM)_VN_*.pdf` | 쇼피 베트남 |
| `라자다_*.pdf` | 라자다 |
| `큐텐재팬_*.pdf` | 큐텐재팬, STEP 2 수동 입력 |

쇼피는 업체명과 무관하게 `_MY_`, `_PH_`, `_SG_`, `_TH_`, `_TW_`, `_VN_`, `_BR_`, `_MX_` 국가코드 패턴도 함께 인식합니다.

## 로컬 실행

```bash
pip install -r requirements.txt
streamlit run app.py
```

## Streamlit Cloud 배포

GitHub에 전체 폴더를 올리고 Streamlit Community Cloud에서 `app.py`를 메인 파일로 지정합니다.
`packages.txt`에는 Selenium fallback용 Chromium 패키지가 들어 있습니다.

## 양식 파일

매출집계 파일은 템플릿 없이 생성합니다.

다만 업로드용 서식 일치를 위해 아래 양식은 `forms/` 폴더에 포함했습니다.

- `영세율첨부서류명세서 양식.xlsx`
- `수출실적명세서 양식.xlsx`

해당 파일이 없어도 간단 양식으로 생성은 되지만, 업로드용 서식 일치가 중요하면 `forms/` 폴더에 두는 것을 권장합니다.

## .xls 관련

Streamlit Cloud에서는 Windows Excel COM 자동화가 불가능하므로 기본 출력은 `.xlsx`입니다.
`.xls`가 꼭 필요하면 온라인 변환 사이트보다 다음 방식이 안전합니다.

1. 로컬 Windows + Excel 자동화
2. 서버에 LibreOffice 설치 후 headless 변환
3. 단순 값 전용이면 `xlwt`로 별도 `.xls` 생성

세무자료/고객자료가 포함될 수 있으므로 외부 온라인 변환 사이트에 파일을 업로드하는 방식은 권장하지 않습니다.

## v32 변경사항

- 매출집계 엑셀 생성 시 실제 입력 데이터가 있는 시트만 생성합니다.
- 쇼피 PDF만 있으면 쇼피 관련 시트, 해당 통화 시트, 해당 환율 시트만 남깁니다.
- 라자다 PDF가 없으면 `라자다(소포수령증)`, `라자다(MYR)` 등 라자다 관련 시트를 생성하지 않습니다.
- 큐텐재팬 수동 입력이 없으면 `큐텐(소포수령증)`과 `JPY` 시트를 생성하지 않습니다.
- 환율 시트도 실제 사용 통화만 생성합니다. 예: MYR/PHP만 사용하면 `환율(MYR)`, `환율(PHP)`만 생성됩니다.
- 통화명만으로 된 시트도 실제 사용 통화만 생성합니다. 예: `MYR`, `PHP`만 사용하면 `TWD`, `BRL`, `MXN`, `JPY` 등은 생성하지 않습니다.


## v32 변경사항

- 환율 캐시가 요청 기간의 주말/공휴일 날짜까지 직전 영업일 환율로 채워 저장되도록 개선했습니다.
- 같은 기간을 다시 처리할 때 서울외국환중개를 반복 조회하지 않고 `data/exchange_rate_cache.csv`의 저장 환율을 우선 사용합니다.
- 환율 로그에 저장된 환율 사용/부족한 환율 수집 여부가 간단히 표시됩니다.
